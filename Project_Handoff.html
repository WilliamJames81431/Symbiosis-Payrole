Database Architecture (localStorage)

The LocalDatabase class (~line 60 in app.js) simulates a full relational DB using named localStorage keys. All CRUD is through helper methods on this class.

localStorage KeyData StoredStructure
symbiosis_organizationsAll org recordsArray of org objects
symbiosis_employeesAll employees across all orgsArray of employee objects

symbiosis_payroll_ledgerLocked payroll records{org_id: {'YYYY-MM': {status, records}}}
symbiosis_schemasCSV column mappingsNested by org ID

Payroll Calculation Engine

Core function: calculatePayrollForEmployee() (~line 400, app.js)

A
Determine Working Days
Calendar days in month. Checks doj (joined mid-month → prorate from that date). Checks exit_date (left mid-month → prorate to that date). Counts P + EL + CO days from attendance. Payable = min(eligible, worked).
B
Calculate Gross Salary
Basic = CTC × basic_pct%  ·  HRA = Basic × 40%  ·  Special Allowance = CTC - Basic - HRA  ·  Gross = (CTC ÷ calendar_days) × payable_days + OT_pay
C
Statutory Deductions (India)
EPF: 12% of Basic, capped ₹1,800/month (wage ceiling ₹15,000)  ·  ESI: 0.75% if Gross ≤ ₹21,000  ·  PT: State-specific slabs from PT_TABLES (5 states)  ·  TDS: Fixed rate or auto-computed annual TDS from slabs
D
HRA Exemption — Section 10(13A)
Exempt = min(actual_HRA_received, basic × 50% metro / 40% non-metro, rent_paid − basic × 10%)
E
Net Pay
Net = Gross − EPF − ESI − PT − TDS + sandboxAdjustments[emp_id]

Global State Object

const state = {
  isLoggedIn: false,
  currentRole: null,           // 'ERP' | 'HR' | 'Employee'
  currentOrgId: null,          // e.g. 'org_tata'
  currentUserId: null,         // e.g. 'EMP101' (for Employee role)
  activeTab: null,             // current active sidebar tab name
  activeMonthYear: '2026-06',  // YYYY-MM — drives all payroll calc
  currentTheme: 'classic-corporate',
  sandboxAdjustments: {},      // { emp_id: { amount, reason } }
}

Saved to localStorage('symbiosis_session') on every meaningful change. Restored on page load before first render. activeMonthYear is the single source of truth for which month's payroll is being processed.

Rendering Architecture

Manual SPA — no router library. Navigation = DOM replacement via innerHTML.

// Entry Point
window.onload → loadSessionState() → renderCurrentView()

// renderCurrentView() decision tree:
if (!state.isLoggedIn)           → showLoginScreen() + renderLoginPage()
if (state.currentRole === 'ERP') → showAppShell() + buildSidebar() + renderERPTab()
if (state.currentRole === 'HR')  → showAppShell() + buildSidebar() + renderHRTab()
if (state.currentRole === 'Emp') → showAppShell() + buildSidebar() + renderEmployeeTab()

// renderMainContent() — enhanced with skeleton loaders:
step 1: inject shimmer skeleton HTML into document.getElementById('app-body')
step 2: setTimeout(150ms)
step 3: call actual render function → replaces skeleton with real content

// Tab switching:
switchTab(tabName) → state.activeTab = tabName → saveSession() → renderCurrentView()

Canvas Charts (Interactive)

🍩
Department Donut Chart
Draws ring arcs per department. Mousemove hit-tests using distance-from-center + angle math. Hovered arc expands +5px radius and +4px line width. Floating tooltip at cursor. Animates from 0° to full over 20 frames (animProgress += 0.05).
📊
Attendance Bar Chart
Counts P/A/EL/WO across all employees. 4 color-coded vertical bars. Mousemove checks bounding rect of each bar. Hovered bar gets glowing shadow. Bars animate growing upward from 0 height over 20 frames. Shared tooltip div.

Both charts share one div#chart-tooltip appended to document.body, positioned at e.clientX / e.clientY via absolute positioning.

Premium UI Features Added

🌙
Dark / Light Mode Toggle
Button in topbar (moon/sun icon). toggleTheme() flips data-theme on <html>. Saved to localStorage. initTheme() restores preference before first render. Charts re-draw with correct colors after toggle.
🔍
Spotlight Search (Ctrl+K)
Global keyboard shortcut. Blurred glass overlay. Searches employees by name/ID + quick actions. Each result is clickable and executes immediately. ESC closes. handleSpotlightInput() → renderSpotlightResults() on each keystroke.
💀
Skeleton Loaders
renderMainContent() first injects shimmer placeholder divs (150ms), then replaces with real content. Uses CSS skeletonShimmer gradient animation. Makes tab switching feel fast and native.
🔔
macOS-Style Stacking Toasts
showNotificationToast(title, msg, type) creates toasts that stack in top-right. Colored left-border accent, icon, auto-dismiss after 4s. toastIn slide+scale in, toastOut slide out with height collapse.
📈
Animated Canvas Charts
Both charts animate on draw (20-frame easing). Hover triggers floating tooltip positioned at cursor. Hovered elements expand/glow. All via shared div#chart-tooltip on document.body.
✨
Animation Suite
Sidebar nav: translateX(4px) on hover. Org cards: translateY(-2px) lift. Stat cards: staggered fadeUp entrance (nth-child delays). Bell dot: infinite bellPulse. Theme toggle: rotates 20° on hover. Day badges: scale(1.2) on hover. prefers-reduced-motion guard added.

Attendance CSV Upload Wizard

1
Upload Step
Drag-and-drop zone or click to browse. Accepts .csv files. Parses using JavaScript's FileReader API.
2
Column Mapping
Maps CSV columns → system fields: Employee ID, Name, D1–D31 day columns, OT hours. Schema is persisted per-org in localStorage so next upload auto-maps columns.
3
Name Mismatch Resolution
If CSV employee names don't match DB names, shows a side-by-side comparison table and asks HR to confirm or override each mismatch.
4
Import
Saves mapped attendance object to symbiosis_attendance[org_id][YYYY-MM]. Confirmation toast shown. Tab refreshes with new data.

Payroll Sandbox

  • Shows calculated Gross, EPF, ESI, PT, TDS, Net for every employee in a table
  • Inline adjustment fields — add bonus or deduction with reason text per employee
  • Warning flags for: mid-month joiners, leavers, ESI-eligible employees, missing attendance data
  • Approve & Lock button — freezes payroll, status → 'Locked', record moves to Ledger
  • Locked payroll is immutable — HR can only view, not edit
